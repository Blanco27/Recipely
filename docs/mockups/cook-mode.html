Verhalten

Kochmodus

  • Start über einen schwebenden „🍳 Kochen"-FAB im Detail-Screen (nur wenn Schritte vorhanden).
  • Ein Schritt pro Bildschirm, wischen oder „Weiter"/„‹" zum Navigieren.
  • Display bleibt an (keep-screen-on), Hochformat, große Fraunces-Schrift.
  • Letzter Schritt → Abschluss-Screen „Guten Appetit".
  • Schritt-Bilder inline; kein Zutaten-Sheet (bewusst fokussiert).
Timer

Aus Schritttext

  • Dauer wird im Text erkannt: „20 Minuten / 20 min / 1 Std / 1 h" (DE + EN).
  • Antippbare Timer-Pille; läuft als Foreground-Service weiter.
  • Benachrichtigung + Ton/Vibration bei Ablauf (POST_NOTIFICATIONS).
  • Tippen pausiert/fortsetzt; ein aktiver Timer zur Zeit.
Technik

Geringes Risiko

  • 100% Compose + Material 3, neuer Screen, neue Route cook/{id}.
  • Kein Datenmodell-Änderung — nutzt vorhandene Steps + Bilder.
  • Neuer TimerService + Notification-Channel; DE/EN-Strings.
  • Umsetzung muss exakt diesem Mockup entsprechen.
