This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project state

Recipely is a native **Android app** (Kotlin) that is currently a fresh Android Studio scaffold. There is **no application code yet** — `app/src/main/` contains only resources and an `AndroidManifest.xml`, and the manifest declares an `<application>` with **no `<activity>`**. The only Kotlin files are the auto-generated example test stubs. The app will not launch until a launcher Activity is created and registered in the manifest (with an `intent-filter` for `MAIN`/`LAUNCHER`).

## Commands

This is a Gradle project; use the wrapper. On Windows/PowerShell use `.\gradlew.bat`; the examples below use `./gradlew`.

```bash
./gradlew assembleDebug              # Build the debug APK
./gradlew build                      # Full build (compile + lint + unit tests)
./gradlew test                       # Run JVM unit tests (app/src/test)
./gradlew testDebugUnitTest          # Unit tests, debug variant only
./gradlew connectedAndroidTest       # Run instrumented tests (needs a device/emulator)
./gradlew installDebug               # Build + install on a connected device/emulator
./gradlew lint                       # Android Lint
./gradlew clean                      # Delete build outputs
```

Run a single unit test (Gradle test filtering):

```bash
./gradlew test --tests "com.nwe.recipely.ExampleUnitTest"
./gradlew test --tests "com.nwe.recipely.ExampleUnitTest.addition_isCorrect"
```

- **JDK 11** is required (`sourceCompatibility`/`jvmTarget` are both 11).
- `local.properties` (git-ignored) must point `sdk.dir` at the Android SDK; it is generated by Android Studio.

## Build configuration

- **Application/namespace id:** `com.nwe.recipely`
- **SDK levels:** `minSdk 24`, `targetSdk 36`, `compileSdk 36`
- **Toolchain:** Kotlin 2.0.21, Android Gradle Plugin 8.13.2, Gradle 8.13

Dependency versions and plugin aliases are centralized in the **Gradle version catalog** at `gradle/libs.versions.toml`. Add or bump dependencies there (under `[versions]` / `[libraries]` / `[plugins]`) and reference them in `app/build.gradle.kts` as `libs.*` — do not hardcode versions in the build scripts.

Currently wired up: `androidx.core:core-ktx`, `androidx.appcompat`, `com.google.android.material` (Material Components), plus JUnit 4 and Espresso for tests. There is no Jetpack Compose, view-binding, or DI framework configured yet — the project uses the classic Android Views + AppCompat + Material stack.
